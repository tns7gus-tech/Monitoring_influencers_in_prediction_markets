# Polymarket 인플루언서 모니터링 서비스 백테스트 계산 정의서

## 1. 문서 목적

이 문서는 Polymarket 상위 트레이더 추적 서비스에서 사용하는 **카피 베팅 백테스트 로직**을 정의한다.  
목표는 다음 질문에 일관된 방식으로 답하는 것이다.

- 특정 트레이더를 과거에 따라갔다면 얼마를 벌었는가?
- 언제부터 따라갔을 때 성과가 어땠는가?
- 단순 승률이 아니라 위험 대비 수익은 어땠는가?
- 여러 트레이더를 동시에 추종하면 성과가 개선되는가?

본 문서는 개발/데이터/제품 관점에서 동일한 계산 기준을 사용하기 위한 기준 문서이다.

---

## 2. 백테스트 범위

### 2.1 포함 범위
- 공개 수집 데이터 기반의 과거 거래 재현
- 트레이더 단일 추종 전략
- 다중 트레이더 바스켓 전략
- 고정 금액 기반 추종
- 비율 기반 추종
- 추종 시작일 기준 성과 비교
- 수수료/슬리피지 반영
- 미청산 포지션의 평가손익 반영

### 2.2 제외 범위
- 완전한 실시간 체결 엔진 재현
- 모든 주문장 레벨의 체결 우선순위 완벽 복원
- 체결 지연이 극단적으로 짧은 HFT 수준 시뮬레이션
- 사용자별 세금 계산
- 실제 폴리마켓 계정별 잔고 이동의 완전 복원

## 3. 핵심 용어 정의

### 3.1 리드 트레이더 (Lead Trader)
모니터링 및 추종 대상이 되는 지갑 주소 또는 사용자 식별자.

### 3.2 팔로워 (Follower)
백테스트 상에서 리드 트레이더를 따라 거래하는 가상 사용자.

### 3.3 추종 이벤트 (Copy Event)
리드 트레이더의 특정 거래를 감지하고, 동일 방향의 진입/청산을 시뮬레이션하는 사건.

### 3.4 시장 (Market)
Polymarket 내 개별 예측 시장. YES/NO 또는 상응하는 outcome 구조를 가질 수 있다.

### 3.5 체결 가격 (Execution Price)
팔로워가 실제로 거래했다고 가정하는 가격. 리드 트레이더 체결가와 동일하지 않을 수 있다.

### 3.6 슬리피지 (Slippage)
리드 트레이더 체결가 대비 팔로워 체결가의 불리한 가격 차이.

### 3.7 실현 손익 (Realized PnL)
청산 완료된 포지션에서 확정된 손익.

### 3.8 평가 손익 (Unrealized PnL)
아직 청산되지 않은 포지션의 현재가 기준 손익.

### 3.9 추종 시작일 (Follow Start Date)
팔로워가 해당 전략을 시작했다고 가정하는 기준 시점.

### 3.10 신호 지연 (Signal Delay)
리드 트레이더 거래 발생 후 팔로워가 이를 인지하고 진입하기까지의 시간 지연.

## 5. 백테스트 전략 유형

## 5.1 단일 트레이더 추종
한 명의 리드 트레이더 거래를 그대로 따라가는 전략.

예시:
- 트레이더 A의 신규 진입만 추종
- 트레이더 A의 진입과 청산 모두 추종
- 트레이더 A의 YES 매수만 추종

## 5.2 다중 트레이더 바스켓 추종
여러 트레이더의 신호를 집계해 추종하는 전략.

예시:
- 최근 90일 수익률 상위 10명 추종
- 카테고리별 상위 3명만 추종
- 동일 시장에 3명 이상 동시 진입 시만 추종

## 5.3 조건부 추종
특정 조건 만족 시에만 추종.

예시:
- 거래 금액이 500달러 이상일 때만 추종
- 시장 잔여 기간이 24시간 이상 남은 경우만 추종
- 리드 트레이더 최근 30일 샤프 유사 지표 상위일 때만 추종

---

## 6. 추종 판단 규칙

## 6.1 신규 진입 정의
아래 조건 중 하나를 만족하면 신규 진입으로 간주한다.

1. 해당 트레이더가 해당 시장/아웃컴 포지션을 보유하지 않다가 처음 매수
2. 직전 보유 수량이 0에 수렴했고 이후 다시 매수
3. 방향이 바뀐 후 최초 진입

### 구현 권장 기준
- `position_before == 0` 이고 `buy size > 0` 이면 신규 진입
- 최소 수량 임계값을 두어 먼지 거래는 제외

예시:
- 0주 보유 상태에서 YES 100주 매수 → 신규 진입
- YES 100주 보유 상태에서 YES 20주 추가 매수 → 추가 진입
- YES 100주 보유 후 전량 매도, 이후 다시 YES 30주 매수 → 신규 진입

## 6.2 추가 진입 정의
이미 동일 방향 포지션을 보유한 상태에서 같은 방향으로 수량을 늘리는 거래.

### 옵션
- **MVP**: 신규 진입만 추종
- **확장형**: 추가 진입도 비례 추종

## 6.3 부분 청산 정의
보유 포지션 일부만 매도한 경우.

## 6.4 완전 청산 정의
시장/아웃컴 기준 순보유 수량이 0이 된 경우.

## 6.5 반대 방향 전환 정의
한 포지션을 청산한 직후 또는 사실상 동일 거래 흐름에서 반대 outcome 으로 진입하는 경우.

예시:
- YES 포지션 전량 청산 후 NO 매수

### 처리 규칙
반대 방향 전환은 아래 두 이벤트로 분리한다.
1. 기존 포지션 청산
2. 새로운 방향 신규 진입

---

## 7. 포지션 복원 규칙

리드 트레이더의 누적 거래로부터 시장별 포지션 상태를 복원한다.

## 7.1 기본 원칙
- 시장 + outcome + trader 단위로 수량과 평균단가를 추적
- buy는 수량 증가
- sell은 수량 감소
- sell 시 평균단가 기준 실현손익 계산

## 7.2 평균단가 방식
기본은 **가중평균단가 방식**을 사용한다.

### 계산식
- 신규 평균단가 = `(기존보유수량 × 기존평균단가 + 신규매수수량 × 신규매수가격) / 총보유수량`

### 선택 대안
- FIFO
- LIFO

### 권장
- 서비스 표시는 가중평균단가 기본
- 내부 검증에서는 FIFO 비교 가능

## 8. 팔로워 주문 생성 규칙

## 8.1 추종 트리거
리드 트레이더 거래가 감지되면 아래 순서로 판단한다.

1. 추종 대상 트레이더인지 확인
2. 전략 필터 충족 여부 확인
3. 신규/추가/청산 거래인지 분류
4. 추종 가능 시간 조건 확인
5. 팔로워 주문 생성

## 8.2 진입 시점
리드 거래 발생 시각 `t_lead` 이후, 팔로워 진입 시각 `t_follow`는 다음과 같이 정의한다.

`t_follow = t_lead + signal_delay`

### signal_delay 예시
- 0초: 이상적 추종
- 5초: 빠른 감지
- 30초: 현실적 웹훅/폴링 기반
- 5분: 알림 보고 수동 진입 시뮬레이션

## 8.3 가격 결정 우선순위
팔로워 체결가격은 아래 우선순위로 계산한다.

1. `t_follow` 시점의 best ask / best bid
2. 해당 시점 근접 체결가
3. mid price + 슬리피지 모델
4. 마지막 유효 가격

### 매수 시
- 기본: ask 사용
- ask 부재 시: mid + slippage

### 매도 시
- 기본: bid 사용
- bid 부재 시: mid - slippage

---

## 9. 주문 크기 결정 방식

## 9.1 고정 금액 방식
각 추종 이벤트마다 일정 금액을 사용.

예시:
- 이벤트당 20달러
- 이벤트당 100달러

### 장점
- 구현 단순
- 비교 용이

### 단점
- 리드 트레이더 규모 반영이 약함

## 9.2 비율 추종 방식
리드 트레이더 거래 금액의 일정 비율만큼 추종.

예시:
- 리드 거래 금액의 10%
- 리드 거래 수량의 5%

### 장점
- 리드의 conviction 반영 가능

### 단점
- 고래의 대형 주문에 과도하게 연동될 수 있음

## 9.3 포트폴리오 비중 방식
전체 자본 대비 최대 비중을 기준으로 주문 크기를 제한.

예시:
- 시장당 최대 5%
- 트레이더당 최대 20%
- 동일 이벤트군 총합 25% 이하

## 9.4 권장 기본 설정
MVP 기준 권장:
- 주문 크기: 이벤트당 고정 50달러
- 시장당 최대 익스포저: 200달러
- 동일 트레이더 일일 최대 사용금액: 500달러

## 10. 수수료 및 슬리피지 모델

## 10.1 수수료
공식 수수료 체계가 확인되면 실제 값을 사용하고, 그 전까지는 설정값 기반으로 계산한다.

### 계산식
- 매수 수수료 = `notional × fee_rate_buy`
- 매도 수수료 = `notional × fee_rate_sell`

### 기본 설정 예시
- `fee_rate_buy = 0.0 ~ 0.2%`
- `fee_rate_sell = 0.0 ~ 0.2%`

## 10.2 슬리피지
슬리피지는 다음 세 가지 모델 중 하나를 사용한다.

### A. 고정 bp 모델
- `execution_price = reference_price × (1 + slippage_bp)` for buy
- `execution_price = reference_price × (1 - slippage_bp)` for sell

예시:
- 20bp = 0.20%

### B. 유동성 기반 모델
주문 금액이 시장 유동성 대비 클수록 슬리피지를 크게 부여.

예시:
- `slippage = base + k × (order_amount / depth_estimate)`

### C. 스냅샷 주문장 기반 모델
orderbook depth snapshot 이 있으면 실제 체결가능 가격을 누적 계산.

## 10.3 권장 단계별 적용
- 1단계: 고정 bp 모델
- 2단계: 유동성 기반 모델
- 3단계: 주문장 깊이 기반 모델

## 11. 청산 규칙

## 11.1 리드 청산 추종
리드 트레이더가 포지션을 줄이거나 전량 청산하면 팔로워도 동일 비율로 청산.

### 예시
- 리드가 100주 보유 중 40주 매도 → 팔로워도 보유량의 40% 매도
- 리드가 전량 청산 → 팔로워 전량 청산

## 11.2 만기 청산
시장 종료 또는 결산 시점에 보유 포지션을 자동 정산한다.

### 규칙
- 승리 outcome 보유 시 payoff = 1.0 기준 정산
- 패배 outcome 보유 시 payoff = 0.0 기준 정산
- 수수료가 있다면 정산 수수료 반영

## 11.3 시간 기반 강제 청산
선택적 전략 규칙.

예시:
- 시장 종료 1시간 전에는 신규 진입 금지
- 시장 종료 10분 전 일괄 청산

## 11.4 손절/익절 규칙
전략 옵션으로 제공 가능.

예시:
- 진입 후 -30% 손실 시 청산
- +50% 수익 시 절반 청산

---

## 12. 미청산 포지션 평가 규칙

백테스트 종료 시 보유 중인 포지션은 평가손익으로 계산한다.

## 12.1 평가 기준 가격
우선순위는 다음과 같다.

1. 백테스트 종료 시점의 mid price
2. 최근 체결가
3. 시장 종료 시 정산가

## 12.2 계산식
- `unrealized_pnl = (mark_price - avg_entry_price) × quantity - fees`

## 13. 자본 관리 규칙

## 13.1 초기 자본
전략별 초기 자본을 명시한다.

예시:
- 1,000달러
- 10,000달러

## 13.2 가용 현금 차감
매수 시 notional + fee 만큼 현금을 차감.

## 13.3 현금 부족 처리
아래 중 하나를 전략 옵션으로 둔다.

1. 주문 스킵
2. 가능한 범위까지만 축소 체결
3. 우선순위 규칙에 따라 일부만 실행

### 권장 기본
- 현금 부족 시 주문 축소 체결
- 최소 주문 금액 미만이면 스킵

## 13.4 익스포저 제한
- 시장당 최대 비중
- 트레이더당 최대 비중
- 카테고리당 최대 비중
- 동시 보유 시장 수 제한

## 14. 성과 지표 정의

## 14.1 기본 지표
- 누적 수익률
- 누적 손익(USD)
- 실현 손익
- 평가 손익
- 승률
- 평균 이익 거래 수익
- 평균 손실 거래 손실
- Profit Factor
- 최대 낙폭(MDD)
- Sharpe 유사 지표
- Sortino 유사 지표
- 거래 수
- 평균 보유 기간

## 14.2 계산식 예시

### 누적 수익률
`total_return = (ending_equity - starting_equity) / starting_equity`

### 승률
`win_rate = profitable_closed_trades / total_closed_trades`

### Profit Factor
`profit_factor = gross_profit / abs(gross_loss)`

### 최대 낙폭
`MDD = max((peak_equity - trough_equity) / peak_equity)`

## 14.3 최근 구간 지표
- 최근 7일
- 최근 30일
- 최근 90일
- 연도별

## 14.4 리스크 조정 지표의 필요성
단순 승률은 왜곡 가능성이 크다.

예시:
- 90% 승률인데 한 번 크게 손실 보는 전략
- 소액만 자주 이기고 대형 손실을 보는 전략

따라서 리더보드에는 최소 아래 4개를 함께 노출한다.
- 누적 수익률
- 최근 30일 수익률
- MDD
- 거래 수

## 15. 리더보드용 점수 정의

단순 수익률 정렬 대신 가중 점수를 사용한다.

## 15.1 추천 점수 예시

`score = A × normalized_return + B × normalized_win_rate + C × normalized_profit_factor - D × normalized_mdd + E × normalized_sample_size`

### 기본 가중치 예시
- A = 0.35
- B = 0.15
- C = 0.20
- D = 0.20
- E = 0.10

## 15.2 최소 표본 조건
리더보드 노출 최소 조건 예시:
- 최근 90일 닫힌 거래 20건 이상
- 총 거래금액 1,000달러 이상
- 추적 기간 30일 이상

## 16. “몇 일부터 따라갔으면” 기능 정의

## 16.1 목적
사용자가 특정 시작일을 선택했을 때, 그 시점부터 동일 전략을 적용한 성과를 계산한다.

## 16.2 시작일 처리 규칙
`follow_start_date` 이전 거래는 팔로워가 추종하지 않는다.

### 중요한 원칙
시작일 이전에 리드 트레이더가 이미 보유하고 있던 포지션은 기본적으로 **따라가지 않은 것**으로 처리한다.

즉:
- 시작일 이후 발생한 신규 신호만 추종
- 시작일 이전부터 열려 있던 포지션은 무시

## 16.3 옵션: 기존 포지션 승계 모드
확장 기능으로 제공 가능.

### 방식
시작일 시점의 리드 보유 포지션을 당시 시장가격으로 가상 복제한다.

### 주의
이 모드는 가정이 더 강하므로 기본값으로 두지 않는다.

## 16.4 사용자 UI 표시 문구 예시
- 기본 모드: “선택한 날짜 이후 발생한 신규 거래만 추종”
- 승계 모드: “선택한 날짜 시점의 기존 보유 포지션도 복제 포함”

## 18. 데이터 품질 및 예외 처리

## 18.1 중복 거래 제거
동일 trade_id 또는 동일 tx_hash + index 조합은 중복 제거.

## 18.2 누락 데이터 처리
가격 스냅샷 누락 시 최근 유효 값 보간 사용.  
단, 보간 사용 여부를 로그로 남긴다.

## 18.3 비정상 거래 필터
- size = 0
- price <= 0 또는 >= 1을 벗어나는 오류값
- 시스템 복원 실패 데이터

## 18.4 포지션 불일치 처리
누적 포지션이 음수로 내려가는 경우:
1. 해당 거래를 오류 후보로 표기
2. 가능한 경우 이전 거래 재정렬
3. 해결 불가 시 해당 시장 포지션 복원 중단 후 confidence 낮춤

## 24. 권장 MVP 계산 규칙 요약

1. 신규 진입만 추종
2. 리드 청산은 비례 추종
3. 시작일 이전 포지션은 복제하지 않음
4. 주문 크기는 이벤트당 고정 금액
5. 체결가는 `t_follow` 시점 ask/bid 우선
6. 가격 부재 시 mid ± 고정 slippage bp
7. 수수료는 설정값 기반 반영
8. 미청산 포지션은 종료 시점 mark price 평가
9. 리더보드는 수익률 단독이 아니라 MDD/표본수 포함 점수 사용
10. 결과 화면에는 항상 가정값과 한계를 명시

## 26. 부록: 간단한 손익 예시

예시 조건:
- 초기 자본: 1,000달러
- 주문 크기: 100달러
- 수수료: 0.1%
- 슬리피지: 0.2%

리드 트레이더가 다음과 같이 거래했다고 가정한다.

1. 2026-01-10 10:00, YES 매수, 가격 0.40
2. 2026-01-12 15:00, YES 전량 매도, 가격 0.70

팔로워는 30초 지연 후 진입/청산했다고 가정한다.

### 팔로워 진입
- 기준가: 0.40
- 매수 슬리피지 반영가: 0.4008
- 100달러로 매수 가능한 수량: 약 249.5주 전후
- 매수 수수료 차감

### 팔로워 청산
- 기준가: 0.70
- 매도 슬리피지 반영가: 0.6986
- 매도 수수료 차감

### 결과
- 진입가와 청산가 차이로 매매차익 발생
- 여기서 양방향 수수료 차감 후 순손익 계산

이 예시는 서비스 내 계산 설명 툴팁으로도 재사용 가능하다.
